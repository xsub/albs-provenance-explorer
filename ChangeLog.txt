ChangeLog
=========

the *how* of what shipped. Paired with `docs/decisions.md` (the *what* and
*why* of each architectural / contract decision): every entry here names the
commit hash and the matching D-entry or F-finding so a reader can follow
either thread.

Conventions:
  - Newest at the top.
  - One bullet per commit; short hash at the start; D-entry / F-finding at end.
  - Pre-F1 history (D1-D44 and the original `main` branch commits) is not
    duplicated here; those choices are captured in `docs/decisions.md` and the
    corresponding code landed before this changelog was instituted. Walk it
    via `git log main --oneline` if needed.

- 514aaab  feat(resolvers): pip / Maven / npm join Go + Cargo behind the
           DependencyResolver contract. PypiResolver uses
           `pip install --dry-run --quiet --no-input --report - -r REQS`
           (pip >= 22.2; the dry-run + JSON report is the whole point) and
           reads each install entry's metadata.name/version. MavenResolver
           uses `mvn -B -q dependency:list -DincludeScope=runtime` and
           parses each `[INFO] groupId:artifactId:packaging:[classifier:]

           type, not identity). NpmResolver uses `npm ls --json --all` and
           recursively walks `dependencies`, dedup'ing on (name, version)
           so diamond deps collapse; tolerates npm's non-zero exit on
           peer-dep warnings when the tree is still present. All three
           follow the existing GoResolver / CargoResolver pattern exactly:
           injectable Runner, FileNotFoundError -> tool-missing, non-zero
           exit -> UNRESOLVABLE preserving requested specs. resolver_for
           factory routes the three new ecosystems; only Gradle stays on
           NullResolver (deferred -- bigger tooling surface). +12 tests,
           each mocking the native tool's stdout. Suite 272 -> 283. ruff +
           mypy --strict clean; no network. (D75; closes E1 for pip /
           Maven / npm, leaving Gradle as the only remaining ecosystem.)
