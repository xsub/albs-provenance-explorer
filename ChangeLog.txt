ChangeLog
=========

Chronological log of implementation progress and fixes on the `max` branch --
the *how* of what shipped. Paired with `docs/decisions.md` (the *what* and
*why* of each architectural / contract decision): every entry here names the
commit hash and the matching D-entry or F-finding so a reader can follow
either thread.

Conventions:
  - Newest at the top.
  - One bullet per commit; short hash at the start; D-entry / F-finding at end.
  - Pre-F1 history (D1-D44 and the original `main` branch commits) is not
    duplicated here; those choices are captured in `docs/decisions.md` and the
    corresponding code landed before this changelog was instituted. Walk it
    via `git log main --oneline` if needed.

- (this)   fix(adapters): HttpCache cannot store Range-ignoring 200 responses
           (review bug #6). `_requests_range_fetch` now requires 206 or 200
           with `len(body) == end - start + 1`; otherwise raises
           `RpmHeaderFetchError` so `_try_candidates` advances to the next
           mirror and HttpCache never sees the bad body. The CDN-cached
           200-with-correct-length case is still accepted (the bytes are
           correct). +1 regression test (mocked Range-ignoring server: the
           call raises and the cache dir stays empty). Suite 257. (D66;
           review bug #6.)

- 227b997  fix(model+security): dry-run isolation + EvidencePatch capture for
           CPE verification (review bug #1). `ProvenanceGraph.copy()` now
           deep-copies node/edge metadata trees, so an in-place mutation on a
           nested value of the copy (`security_identity`, candidate dicts) no
           longer leaks back into the source. `verify_graph_cpe` works on a
           `deepcopy(identity)` per node and routes the final state through
           `graph.update_metadata(...)`, so `RecordingGraph` captures the CPE
           change in `EvidencePatch.metadata_updates`. +2 regression tests;
           coverage golden byte-identical; suite 256. (D65; review bug #1.)

- fd53019  chore: institute ChangeLog.txt + document the convention in

           `decisions.md` rather than duplicated.

- 3346bf1  feat(net): content-addressed HTTP cache + bounded concurrency.

           740 ms on a single RPM; 254 tests pass on el10. +8 tests. (D63;
           review item #8.)

- 59f53f5  refactor(cli): extract `AnalysisPipeline`; migrate `coverage`
           onto it. New `albs_graph/pipeline.py`: RunSpec, EnrichmentStep
           Protocol + 13 step objects, DEFAULT_STEPS (exact historical
           coverage order), `AnalysisPipeline.run(spec, graph, *,
           dry_run=)` returning PipelineResult (enriched graph + per-step
           results by name + reconciliation + cumulative EvidencePatch via
           RecordingGraph). `coverage_command` builds a RunSpec, runs the
           pipeline, and aliases `result.result(name)` back to the existing
           render variables -- rendering is unchanged. Verified by
           byte-identical coverage golden output on the synthetic fixture.
           +5 pipeline unit tests (mechanics not exercised by golden);
           suite 232. (D57; review item #1 -- coverage migrated.)

- 6f41e4c  feat(reconcile): IdentityMismatch rule. Fires when 2+ claims in
           a group assert the same concrete version with different PURL
           coordinates -- two sources disagree on *what* the dep is, not
           merely its version. Conservative: only on >=2 PURL-bearing
           claims (the common single-PURL group never fires); grouped by
           version so a real version disagreement is left to
           `VersionDriftRule`. Fills the documented IDENTITY_MISMATCH gap.
           +3 tests; suite 227. (D56.)

- 11a6455  feat(model): EvidencePatch + RecordingGraph. `RecordingGraph`
           wraps a `ProvenanceGraph` (shared state) and records writes to
           a typed `EvidencePatch`; works in apply-and-record mode
           (default) and dry-run when constructed over a copy. Sets up
           "what would this run change?" UX + per-step patches for the
           pipeline. (D55; review item #6.)

- edbea36  Docs: prune stale roadmap / limitations claims (current gaps
           only). Quality pass over `docs/plan.md` and `docs/limitations.md`
           so they describe the system as it now is. (D54; review item #10.)

- 841bf7a  refactor(albs): first-class build/task model -- typed
           BuildTaskRef / BuildSourceRef collections in parsed ALBS
           metadata. Adapters no longer rely on representative-build +

- c597ec2  perf(graph): index ProvenanceGraph adjacency + node-type lookups.

           scans to O(1) hash lookups + maintained adjacency. (D52; review

- d2a6694  refactor(reconcile): rule-based reconciliation via

- 84f38c5  fix(native): arch-scoped dnf + rpmgraph matching. `dnf
           repoquery` queries the bare name, so on a multi-arch graph
           every arch node inherited the host arch's resolution; now
           scoped to `name.arch`. `rpmgraph` indexed `name -> node` with
           `setdefault` so only the first arch variant got the claims;
           now indexes `name -> [nodes]` and attaches to every arch
           variant. +2 tests. (D47/F8.)

- de440e6  feat(source): per-source `checkout-source` / `source-evidence`
           selector. Batch builds have many sources; the previous code
           always used the build's representative source. `--package`
           resolves the per-task repo+commit (same attribution as the
           graph builder) and `dataclasses.replace` retargets the
           metadata. +1 test. (D47/F6.)

- d09d279  fix(sbom): NEVRA-exact build-SBOM matching. `enrich_graph_with_
           build_sbom` keyed by `(name, arch)` first-wins -- a merged
           graph with two versions of one package attached the *first*
           same-name component's CPE to every match. Now keys by
           `(name, version-release, arch)`. (D47/F3.)

- c8a1488  fix(identity): split vendor-asserted from NVD-verified CPE.
           Vendor SBOM and NVD dictionary verification both establish
           identity but `cpe_status` now distinguishes them (`vendor_
           asserted` vs `verified`); `identity_strength` reports the
           split so a 1.00 axis stays honest about how it was reached.
           (D46/F4.)

- ee72d34  fix: external review scope-boundary findings (Phase 0 + F1).
           F1 = `artifacts_from_source` reimplemented to use per-task
           attribution (`source_to_artifact_path`) rather than
           `graph.reachable`, so a batch build's per-task source linkage
           is precise. (D45/Phase 0 + F1.)
